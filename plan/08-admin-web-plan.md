# 08 Vue 3 管理端计划

## 1. 目标

实现总部管理端，用于查询基础数据、维护盘点模板、创建月盘任务、查看任务结果。

管理端服务总部用户，不处理门店录入体验。

注意：物料和门店来自基础数据库管理系统，管理端只提供查询和选择，不提供新增、编辑、删除物料和门店。

## 2. 技术建议

- Vue 3
- Vite
- Vue Router
- Pinia
- Axios
- Element Plus 或 Naive UI

如果没有既定 UI 组件库，建议使用 Element Plus，适合后台管理系统。

## 3. 页面结构

```text
src/
  main.ts
  router/
  stores/
  api/
  layouts/
  views/
    material/
    store/
    template/
    task/
    result/
  components/
```

## 4. 路由建议

```text
/materials                    # 物料查询
/stores                       # 门店查询
/templates                    # 盘点模板列表
/templates/:id                # 模板编辑详情
/tasks                        # 月盘任务管理
/tasks/create                 # 创建月盘任务
/tasks/:id                    # 任务详情
/tasks/:id/result             # 盘点结果
```

## 5. API 封装

建议按模块拆分：

```text
src/api/material.ts
src/api/store.ts
src/api/template.ts
src/api/task.ts
```

统一处理：

- baseURL
- 请求头
- 错误提示
- 登录态或模拟用户头
- 分页参数

首期可在请求头中模拟：

```text
X-User-Id
X-Role: admin
```

## 6. 基础数据查询页

### 6.1 物料查询

功能：

- 列表展示。
- 按名称/规格搜索。
- 按状态筛选。
- 查看同步时间。

字段：

- 外部物料 ID。
- 物料名称。
- 规格。
- 单位。
- 分类。
- 状态。
- 最近同步时间。

不提供：

- 新增物料。
- 编辑物料。
- 启停物料。

验收：

- 能查询从基础数据库管理系统同步过来的物料。
- 停用物料可查看，但不可在模板中新增选择。

### 6.2 门店查询

功能：

- 列表展示。
- 按门店名称/编码搜索。
- 按状态筛选。
- 查看同步时间。

字段：

- 外部门店 ID。
- 门店名称。
- 门店编码。
- 状态。
- 最近同步时间。

验收：

- 能查询从基础数据库管理系统同步过来的门店。
- 停用门店可查看，但不可创建新的月盘任务。

## 7. 盘点模板页

功能：

- 模板列表。
- 创建模板。
- 编辑模板名称。
- 发布/停用模板。
- 模板详情。
- 新增分区。
- 删除分区。
- 添加物料到分区。
- 删除分区物料。
- 调整物料顺序。

模板详情展示：

- 左侧模板列表。
- 右侧当前模板分区。
- 分区下展示物料列表。
- 物料显示名称、规格、盘点单位。

验收：

- 可创建完整模板。
- 同一分区重复添加同一物料时，页面提示后端错误。
- 停用物料不可添加。
- 物料选择弹窗使用物料查询接口，不在弹窗内创建物料。

## 8. 月盘任务管理页

功能：

- 按月份筛选。
- 按门店筛选。
- 按状态筛选。
- 按模板搜索。
- 创建月盘任务。
- 查看任务详情。
- 查看任务结果。

列表字段：

- 任务名称。
- 门店。
- 月份。
- 模板。
- 截止时间。
- 状态。
- 分区数。
- 物料数。
- 提交时间。
- 操作。

创建任务字段：

- 任务名称。
- 盘点月份。
- 门店范围。
- 应用模板。
- 截止时间。

创建任务时建议展示模板预览：

- 分区数。
- 物料数。
- 分区结构。

验收：

- 总部可以创建任务。
- 门店选择来自门店查询接口。
- 停用门店不可创建任务。
- 创建成功后任务列表出现新任务。
- 可查看任务详情和进度。

## 9. 盘点结果页

功能：

- 展示任务基础信息。
- 按分区查看。
- 按物料汇总查看。

按分区字段：

- 分区名称。
- 物料名称。
- 规格。
- 单位。
- 盘点数量。
- 备注。

按物料汇总字段：

- 物料名称。
- 规格。
- 单位。
- 出现分区数。
- 汇总数量。

验收：

- 已提交任务能查看结果。
- 汇总数量和后端一致。
- 0 数量正常显示。

## 10. 视觉与交互要求

- 管理端应偏工作台风格，信息密度适中。
- 表格、筛选、弹窗、表单要稳定易用。
- 不做营销式首页。
- 页面操作后应刷新局部数据。
- 错误信息应使用后端 message。

## 11. 不做范围

- 不做复杂权限管理页面。
- 不做物料主数据维护。
- 不做门店主数据维护。
- 不做审批流。
- 不做报表大屏。
- 不做数据导出，除非后续明确要求。

```text
6. 实现任务结果页

```
