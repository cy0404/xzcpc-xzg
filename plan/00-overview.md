# 00 项目总览

## 1. 项目定位

盘点工具 1.0 是一套门店月盘数据录入系统。

首期目标：

- 从基础数据库管理系统同步物料和门店主数据。
- 总部维护标准盘点模板。
- 总部创建月盘任务。
- 系统生成任务快照。
- 店长在小程序端按分区录入物料数量。
- 系统按任务维度汇总同一物料跨分区数量。
- 店长提交任务。
- 提交后系统更新门店默认分区物料清单。
- 门店和总部都可以查看盘点结果。

## 2. 技术栈

- 后端：Spring Boot
- 管理端：Vue 3
- 小程序端：原生微信小程序
- 数据库：建议 MySQL 8.x
- 数据库变更：建议 Flyway 或 Liquibase

## 3. 主数据背景

项目已有基础数据库管理系统，那里维护物料和门店的全部信息。

因此盘点工具的定位是：

```text
基础数据库管理系统
  -> 物料主数据
  -> 门店主数据

盘点工具
  -> 同步并引用物料
  -> 同步并引用门店
  -> 维护盘点模板
  -> 创建盘点任务
  -> 生成任务快照
  -> 保存盘点录入
  -> 汇总结果
  -> 更新门店默认分区物料清单
```

核心原则：

- 基础数据库管理系统是物料和门店的唯一主数据源。
- 盘点工具不新增、不编辑、不删除物料和门店。
- 盘点工具可以保存本地同步缓存，用于查询性能、模板配置和任务快照。
- 门店默认分区物料清单属于盘点业务数据，由盘点工具维护。

## 5. 核心业务闭环

```text
基础系统同步物料和门店
-> 总部维护盘点模板
-> 总部创建月盘任务
-> 系统生成任务快照
-> 店长按分区录入
-> 系统汇总跨分区物料
-> 店长提交
-> 系统锁定任务并更新门店默认清单
-> 门店/总部查看结果
```

## 6. 用户角色

### 总部用户

角色：

- 运营
- 供应链
- 财务

权限：

- 查询物料和门店主数据缓存。
- 维护标准模板。
- 创建月盘任务。
- 查看任务结果。

### 门店用户

角色：

- 店长

权限：

- 查看任务。
- 调整当前任务分区与物料结构。
- 录入盘点数量。
- 查看汇总预览。
- 提交任务。
- 查看结果。

## 7. 核心状态

### 任务状态

```text
not_started    未开始
in_progress    进行中
submitted      已提交
```

### 分区状态

分区状态建议实时计算，不优先落库。

```text
not_started    所有物料均未录入
in_progress    部分物料已录入
completed      全部物料已录入，包括数量为 0
```

### 物料录入状态

```text
not_entered    未录入，input_qty 为 null
entered        已录入，input_qty > 0
zero_entered   已录入，input_qty = 0
```

## 8. 最高优先级规则

- 物料和门店来自基础数据库管理系统。
- 盘点工具不拥有物料和门店主数据维护权。
- `null` 和 `0` 必须严格区分。
- `null` 表示未录入。
- `0` 表示已录入为 0。
- 创建任务时必须生成快照。
- 任务快照生成后，不受模板和门店默认清单后续变化影响。
- 未全部录入不可提交。
- 已提交任务不可编辑。
- 提交后需要生成汇总结果。
- 提交后需要更新门店默认分区物料清单。
- 录入数量为 0 的分区物料关系，需要从门店默认清单移除。

## 9. 开发顺序建议

```text
1. 数据库表结构
2. 后端基础架构
3. 主数据同步与查询模块
4. 模板模块
5. 任务创建与快照
6. 门店端录入 API
7. 提交、汇总、默认清单回写
8. 管理端页面
9. 小程序页面
10. 联调和验收
```
