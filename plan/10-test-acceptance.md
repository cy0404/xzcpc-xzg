# 10 测试与验收计划

## 1. 目标

通过固定测试数据和关键场景，验证盘点工具 1.0 的业务闭环和核心规则。

最高优先级：

- 物料和门店以基础数据库管理系统为主数据源。
- 空值和 0 区分。
- 任务快照独立。
- 跨分区汇总准确。
- 未全部录入不可提交。
- 已提交任务不可编辑。
- 提交后门店默认清单回写正确。

## 3. 后端测试场景

### 3.1 主数据同步与查询模块

- 同步物料成功。
- 同步门店成功。
- 查询物料列表成功。
- 查询门店列表成功。
- 物料外部 ID 唯一。
- 门店外部 ID 唯一。
- 基础系统物料状态能正确映射为盘点工具状态。
- 基础系统门店状态能正确映射为盘点工具状态。
- 查询不存在的物料返回错误。
- 查询不存在的门店返回错误。
- 盘点工具不提供新增、编辑、删除物料的管理能力。
- 盘点工具不提供新增、编辑、删除门店的管理能力。

### 3.2 模板模块

- 创建模板成功。
- 添加分区成功。
- 添加物料到分区成功。
- 同一分区重复添加同一物料失败。
- 同一物料添加到不同分区成功。
- 删除分区物料成功。
- 调整排序成功。
- 模板添加物料时，只能选择同步缓存中的启用物料。

### 3.3 任务快照

- 创建任务成功。
- 停用门店不可创建任务。
- 创建任务后生成 `task_zone`。
- 创建任务后生成 `task_zone_material`。
- 快照保存物料名称、规格、单位。
- 修改模板后，已创建任务不变化。
- 修改基础数据库管理系统中的物料后，已创建任务快照不变化。
- 修改基础数据库管理系统中的门店后，已创建任务基础信息不变化。

### 3.4 分区录入

- 保存整数成功。
- 保存小数成功。
- 保存 0 成功，状态为 `zero_entered`。
- 保存空值成功，状态为 `not_entered`。
- 保存负数失败。
- 已提交任务保存失败。

### 3.5 汇总预览

- 同一物料跨多个分区时，汇总数量正确。
- 空值不参与预览汇总。
- 0 参与预览汇总。
- 存在未录入时，`canSubmit = false`。
- 全部录入后，`canSubmit = true`。

### 3.6 提交任务

- 未全部录入时提交失败。
- 全部录入后提交成功。
- 数量为 0 的物料不阻止提交。
- 提交后任务状态为 `submitted`。
- 提交后写入 `submitted_by` 和 `submitted_at`。
- 提交后生成 `task_material_summary`。
- 重复提交失败或幂等返回已提交。
- 提交后保存分区录入失败。

### 3.7 默认清单回写

- `input_qty > 0` 的分区物料关系保留。
- 新增到任务中的物料，若 `input_qty > 0`，写入门店默认清单。
- `input_qty = 0` 的分区物料关系，从门店默认清单移除。
- 回写默认清单不影响当前任务历史记录。
- 回写默认清单不影响总部模板。
- 回写默认清单不影响基础数据库管理系统中的物料主数据。

## 4. 管理端验收场景

- 总部可以打开物料列表。
- 总部可以打开门店列表。
- 总部可以查询物料。
- 总部可以查询门店。
- 总部不能在盘点工具内新增、编辑、启停物料。
- 总部不能在盘点工具内新增、编辑、启停门店。

- 总部可以给分区添加物料。

- 总部可以查看任务列表。
- 总部可以查看任务详情和进度。
- 总部可以查看提交后的结果。
- 结果页支持按分区和按物料汇总切换。

## 5. 小程序验收场景

- 店长可以看到本门店当前任务。
- 店长可以进入任务详情。
- 任务详情显示总体进度和分区列表。
- 店长可以进入分区盘点页。

- 店长可以留空表示未录入。

- 汇总预览显示完成情况。

- 提交后进入结果页。

## 6. 关键边界用例

### 6.1 空值与 0

数据：

```text
鲜奶 input_qty = null
椰浆 input_qty = 0
```

预期：

- 鲜奶状态为 `not_entered`。
- 椰浆状态为 `zero_entered`。
- 任务不可提交，因为鲜奶未录入。
- 椰浆参与汇总，数量为 0。

### 6.2 跨分区汇总

数据：

```text
前台冷藏区 鲜奶 5000 ml
操作台下方 鲜奶 1800 ml
```

预期：

```text
鲜奶汇总数量 = 6800 ml
出现分区数 = 2
```

### 6.3 默认清单移除

数据：

```text
前台冷藏区 椰浆 input_qty = 0
```

预期：

- 提交后，门店默认清单中移除 `南山万象店 + 前台冷藏区 + 椰浆`。
- 当前任务结果仍保留该记录，数量为 0。

### 6.4 快照解耦

流程：

1. 创建任务。
2. 修改模板，删除某分区物料。
3. 查询已创建任务。

预期：

- 已创建任务的分区和物料不变化。

### 6.5 主数据归属

流程：

1. 从基础数据库管理系统同步物料和门店。
2. 在盘点工具管理端查询物料和门店。
3. 尝试在盘点工具中寻找新增或编辑物料入口。

预期：

- 物料和门店可以查询。
- 物料和门店展示外部系统 ID 或同步时间。
- 盘点工具不提供物料新增和编辑入口。
- 盘点工具不提供门店新增和编辑入口。

## 7. AI 每轮编码后的检查清单

每完成一个模块，要求 AI 输出：

- 修改文件列表。
- 新增接口列表。
- 实现的业务规则。
- 未实现范围。
- 运行命令。
- 测试命令。
- 手工验收步骤。

代码检查重点：

- 是否越界实现其他模块。
- 是否把 `0` 当成空值。
- 是否误把物料和门店当作盘点工具自有主数据。
- 是否遗漏后端校验。
- 是否在提交事务中跨表写入。
- 是否污染任务快照。
- 是否让前端承担本该后端校验的规则。

## 8. MVP 验收闭环

第一轮最小可用闭环：

```text
1. 同步物料。
2. 同步门店。
3. 总部创建模板。
4. 总部创建任务。
5. 系统生成任务快照。
6. 店长查看任务。
7. 店长录入分区。
8. 店长查看汇总。
9. 店长提交任务。
10. 系统生成汇总。
11. 系统回写门店默认清单。
12. 总部查看结果。
13. 门店查看结果。
```
