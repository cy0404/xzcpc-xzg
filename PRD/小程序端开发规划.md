## 一、项目背景

总部端（Vue + Spring Boot）已上线，覆盖物料管理、模板管理、月盘任务下发。本次新增**小程序端**给门店店长使用，承担月盘任务的录入、调整、汇总、提交全流程。

- 后端代码仓：`xzcpc-xzg/`（Java 17 + Spring Boot 3.2 + MyBatis-Plus + MySQL 8.0）
- 设计稿：`design/小程序端/`（12 张）
- PRD：`PRD/盘点工具 1.0 PRD.docx`

## 三、登录与身份方案

### 流程
```
店长扫码进入小程序
   ↓
uni.login() 拿 code
   ↓
POST /api/mp/auth/wx/login  { code }
   ↓ 后端：用 code 换 openid + session_key，落 store_manager_session（未绑定状态）
   ↓ 返回临时 token + 是否已绑定手机号
   ↓
若未绑定 → 显示「获取手机号」button (open-type="getPhoneNumber")
点击 → 拿到 phoneCode
   ↓
POST /api/mp/auth/wx/bind-phone  { phoneCode }
   ↓ 后端：用 phoneCode + session_key 解密手机号
   ↓ 匹配 task.manager_phone（任意未提交任务命中即可）
   ↓ 匹配上 → 更新 session.manager_phone，颁发正式 JWT
   ↓ 匹配不上 → 返回 4031「您还没有被指派盘点任务，请联系总部」
   ↓
进入任务列表
```

### 数据库变更
- `task` 表加 `manager_phone VARCHAR(20)` + 索引
- `task_zone` 表加 `zone_saved TINYINT DEFAULT 0`（店长点过「保存本分区」置 1）
- 新增 `store_manager_session` 表：`openid / unionid / session_key / manager_phone / token / 时间戳`

迁移脚本：`database/migration-mp.sql`（已写）

## 四、后端接口设计（前缀 `/api/mp/`）

### 4.1 鉴权
| 方法 | 路径 | 说明 | 入参 | 出参 |
|---|---|---|---|---|
| POST | `/api/mp/auth/wx/login` | 微信登录换 token | `{ code }` | `{ token, bound:boolean, phone? }` |
| POST | `/api/mp/auth/wx/bind-phone` | 手机号校验绑定 | `{ phoneCode }`（需 Bearer 临时 token） | `{ token, phone, storeName }` |
| POST | `/api/mp/auth/logout` | 退出登录（可选） | - | - |
| GET | `/api/mp/auth/me` | 取当前店长信息 | - | `{ phone, storeId, storeName }` |

### 4.2 任务
| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/mp/tasks` | 店长视角任务列表（按手机号过滤，按状态分组：current + history） |
| GET | `/api/mp/tasks/{id}` | 任务详情，含每个分区的进度（已录入/总数）、`zone_saved` 标记 |
| POST | `/api/mp/tasks/{id}/submit` | 提交任务（校验全部 zone_saved=1）|
| GET | `/api/mp/tasks/{id}/summary` | 盘点汇总（任务完成情况 + 物料跨分区合并） |
| GET | `/api/mp/tasks/{id}/result` | 已提交后只读结果（复用现有 `/api/tasks/{id}/result`） |

### 4.3 分区物料录入
| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/mp/tasks/{id}/zones/{zoneId}/materials` | 分区下物料列表（含数量、状态、备注） |
| POST | `/api/mp/tasks/{id}/zones/{zoneId}/save` | **保存本分区**，请求体 `{ items:[{materialId, qty, remark}] }`，全部填齐且无负数才能保存成功，置 `zone_saved=1` |
| POST | `/api/mp/tasks/{id}/zones/{zoneId}/item-save` | 单条录入（可选，用于失焦自动保存草稿） |

### 4.4 调整分区与物料
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/mp/tasks/{id}/zones` | 新增分区 `{ zoneName }` |
| DELETE | `/api/mp/tasks/{id}/zones/{zoneId}` | 删除分区（若分区下任意物料已录入，前端需先二次确认） |
| PUT | `/api/mp/tasks/{id}/zones/sort` | 分区排序 `{ zoneIds:[] }` |
| POST | `/api/mp/tasks/{id}/zones/{zoneId}/materials` | 加入物料 `{ materialId }` |
| DELETE | `/api/mp/tasks/{id}/zones/{zoneId}/materials/{materialId}` | 从分区移除物料（已录入需二次确认） |
| PUT | `/api/mp/tasks/{id}/zones/{zoneId}/materials/sort` | 物料排序 `{ materialIds:[] }` |
| GET | `/api/mp/tasks/{id}/zones/{zoneId}/candidates` | 可加入物料候选（带 `inCurrentZone:boolean` 标记） |

### 4.5 统一约定
- 全部接口前缀 `/api/mp/`
- Header：`Authorization: Bearer <jwt>`（白名单：登录两个接口、绑定接口）
- 返回体：`{ code, message, data }`（复用现有 `R<T>`）
- 错误码：
  - `401` 未登录或 token 失效
  - `4031` 未匹配到任务（手机号未被指派）
  - `4032` 任务已提交、不可修改
  - `4033` 任务已过期截止时间、不可写
  - `4040` 任务不存在或不属于当前店长
- **截止时间过期**：用 AOP 切面统一拦截所有 `POST/PUT/DELETE /api/mp/tasks/**`，过期返回 4033

## 六、业务规则速查

| 规则 | 说明 |
|---|---|
| 单店长单任务 | 同一手机号同一时间最多一个未提交任务（创建任务时后端校验） |
| 分区状态 | 三态：未开始（无任何录入）/ 进行中（部分录入）/ 已录入（所有物料填齐 + `zone_saved=1`） |
| 录入校验 | 数量必填、可为 0、不可为负、不可为空字符串；负数前端红框 + 阻止保存 |
| 备注 | 点击录入框右侧 ✏️ 弹窗，含数量 + 备注，可选填 |
| 调整分区/物料 | 仅影响当前任务快照，不修改基础物料库和门店默认清单 |
| 删除已录入 | 删除时若该物料已录入数量，必须二次弹窗确认提示「将丢失 X 数据」 |
| 截止时间过期 | 列表/详情红字提示；所有写接口 AOP 拦截返回 4033；仅可查看 |
| 已提交任务 | 不可编辑、不可重复提交、不可删除分区；进入即结果页 |
| 提交后副作用 | 调用现有 task 模块的 submit 逻辑：更新 task_material_summary + store_zone_material（已实现） |

## 九、风险点与待确认

| 风险点 | 当前处理 |
|---|---|
| 微信测试号无法用 button.getPhoneNumber（该接口需要正式认证小程序） | 本地阶段提供 Mock 切换：开发模式下 `bind-phone` 可直接传明文手机号绕过 |
| WxJava 版本与 SpringBoot 3.2 兼容性 | 选 4.6.0 已验证可用 |
| `task` 现有创建逻辑（admin）需要补 manager_phone 字段 | 阶段二完成后开一个补丁任务，修改 admin 的任务创建页 |
| 调整分区/物料是否会影响门店默认清单 store_zone_material？ | **不影响**。提交时根据 task_zone/task_zone_material 重建 store_zone_material（已实现） |
| 同一物料在不同分区，删除单分区中的录入是否影响汇总？ | 汇总实时计算（`task_material_summary` 在保存/提交时增量更新）；删除时也要更新汇总 |
| 备注是否参与汇总展示？ | 汇总页不展示备注；结果页按分区展示备注（已在设计图体现） |
